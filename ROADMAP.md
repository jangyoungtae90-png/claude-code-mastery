# 개발자 웹 이력서 개발 로드맵

이 문서는 개발자 포트폴리오/이력서 웹사이트 구축을 위한 7단계 개발 계획을 정의합니다.

---

## Phase 1: 프로젝트 초기화 및 기본 구조 (1-2일)

### 목표
프로젝트의 기본 틀 구성 및 개발 환경 설정

### 작업 항목
- [ ] Git 저장소 초기화
- [ ] 프로젝트 디렉토리 구조 생성
  - `index.html` — 메인 HTML 템플릿
  - `css/styles.css` — 커스텀 CSS 파일
  - `js/script.js` — JavaScript 파일
  - `assets/images/` — 이미지 디렉토리
  - `assets/pdf/` — 이력서 PDF 디렉토리
- [ ] `index.html` 기본 틀 작성 (Tailwind CDN 포함)
- [ ] `CLAUDE.md` 작성 (프로젝트 가이드라인)
- [ ] `README.md` 작성 (프로젝트 설명)

### 산출물
- 기본 HTML 구조 (메타 태그, 문자 인코딩 등)
- Tailwind CSS CDN 연동
- 프로젝트 문서화

---

## Phase 2: 페이지 레이아웃 및 섹션 설계 (2-3일)

### 목표
웹 이력서의 주요 섹션 구조 완성

### 작업 항목
- [ ] 네비게이션 바 마크업 작성
  - 로고/이름 섹션
  - 네비게이션 링크 (About, Experience, Skills, Projects, Contact)
  - 모바일 메뉴 토글 버튼
  
- [ ] Hero 섹션 작성
  - 프로필 이미지
  - 인사말 텍스트
  - CTA 버튼 (이력서 다운로드, 연락처)
  
- [ ] About 섹션 작성
  - 전문가 요약 (Elevator Pitch)
  - 간단한 소개글
  
- [ ] Experience 섹션 작성
  - 직무 경력 카드 (회사명, 직급, 날짜, 담당 업무)
  - 최소 3개 이상의 경험 예제
  
- [ ] Skills 섹션 작성
  - 기술 스택 분류 (Frontend, Backend, Tools 등)
  - 숙련도 표시 (프로그레스 바 또는 별점)
  
- [ ] Projects 섹션 작성
  - 프로젝트 카드 (제목, 설명, 기술 스택, 링크)
  - 최소 3-4개 예제 프로젝트
  
- [ ] Contact 섹션 작성
  - 이메일 주소
  - SNS 링크 (GitHub, LinkedIn, Twitter 등)
  - 간단한 연락처 폼

- [ ] Footer 작성
  - 저작권 정보
  - 마지막 수정 날짜

### 스타일 적용
- [ ] Tailwind CSS 유틸리티 클래스로 기본 스타일링
- [ ] 컬러 팔레트 정의 (Primary, Secondary, Accent)
- [ ] 타이포그래피 설정 (Google Fonts)
- [ ] 기본 반응형 디자인 (mobile-first)

### 산출물
- 완성된 HTML 마크업 (모든 섹션)
- Tailwind 클래스 기반 스타일링
- 반응형 레이아웃 (모바일/태블릿/데스크톱)

---

## Phase 3: 콘텐츠 작성 및 이미지 준비 (2-3일)

### 목표
실제 이력서 콘텐츠 작성 및 미디어 준비

### 작업 항목
- [ ] Hero 섹션 콘텐츠
  - 이름, 직급, 한 줄 소개
  - 프로필 사진 준비 (원형 또는 정사각형)

- [ ] About 섹션 콘텐츠
  - 전문가로서의 요약 (150-200자)
  - 경력 주요 성과 3-4가지

- [ ] Experience 섹션 콘텐츠
  - 현재/이전 회사, 직급, 기간
  - 각 경험에서의 주요 성과 (Bullet points)
  - 사용 기술/스택

- [ ] Skills 섹션 콘텐츠
  - 프로그래밍 언어 (Python, JavaScript, Java 등)
  - 프론트엔드 기술 (HTML, CSS, React, Vue 등)
  - 백엔드 기술 (Node.js, Django, Spring 등)
  - 도구 및 플랫폼 (Git, Docker, AWS 등)
  - 각 항목별 숙련도 표시

- [ ] Projects 섹션 콘텐츠
  - 프로젝트 제목, 설명, 기간
  - 사용 기술 태그
  - GitHub 또는 라이브 링크
  - 프로젝트 스크린샷 또는 썸네일 준비

- [ ] Contact 섹션 정보
  - 이메일 주소
  - GitHub 프로필 링크
  - LinkedIn 프로필 링크
  - 기타 소셜 미디어

- [ ] 이미지 최적화
  - 프로필 사진 압축 (webp 포맷 권장)
  - 프로젝트 스크린샷 최적화
  - 이미지 파일명 영어로 정규화

- [ ] 이력서 PDF 파일
  - PDF 이력서 작성 및 저장
  - 다운로드 링크 연동

### 산출물
- 모든 섹션의 완성된 콘텐츠
- 최적화된 이미지 파일
- 이력서 PDF 파일

---

## Phase 4: JavaScript 상호작용 및 기능 추가 (2-3일)

### 목표
동적 기능 및 사용자 상호작용 구현

### 작업 항목
- [ ] 네비게이션 기능
  - 네비게이션 링크 클릭 시 부드러운 스크롤 (smooth scroll)
  - 현재 섹션에 따른 활성 링크 하이라이트
  - 모바일 메뉴 토글 (hamburger menu)

- [ ] 스크롤 감지 애니메이션
  - 요소 스크롤 진입 시 페이드 인/슬라이드 인 효과
  - Intersection Observer API 사용 (AOS 라이브러리 선택 가능)

- [ ] 모바일 메뉴 토글
  - 햄버거 아이콘 클릭 시 메뉴 표시/숨김
  - 모바일 메뉴 항목 클릭 시 자동 닫힘
  - 메뉴 애니메이션 (슬라이드 또는 페이드)

- [ ] 다크 모드 준비 (선택사항)
  - 다크 모드 토글 버튼 추가
  - localStorage에 사용자 선호도 저장

- [ ] 간단한 폼 처리 (선택사항)
  - Contact 폼 유효성 검사
  - 폼 제출 이벤트 처리

- [ ] 성능 최적화
  - 이미지 lazy loading 구현
  - JavaScript 최소화 (필요시)

### 산출물
- `js/script.js` — 모든 상호작용 기능 포함
- 부드러운 사용자 경험
- 모바일 메뉴 완전 작동

---

## Phase 5: 반응형 디자인 및 UI/UX 개선 (2-3일)

### 목표
모든 디바이스에서 최적의 사용자 경험 제공

### 작업 항목
- [ ] 반응형 디자인 검증
  - 모바일 (320px 이상)
  - 태블릿 (768px 이상)
  - 데스크톱 (1024px 이상)
  - 초대형 화면 (1280px 이상)

- [ ] 모바일 최적화
  - 터치 친화적 버튼 크기 (최소 48px)
  - 가독성 좋은 폰트 크기
  - 충분한 여백 (padding, margin)

- [ ] 다크 모드 구현 (선택사항)
  - Tailwind `dark:` 클래스 활용
  - 시스템 테마 감지 및 적용

- [ ] 접근성 개선
  - 시맨틱 HTML 검증 (`<nav>`, `<header>`, `<main>`, `<footer>`)
  - 이미지 alt 텍스트 추가
  - 색상 대비도 확인 (WCAG 표준)
  - 키보드 네비게이션 지원

- [ ] 애니메이션 및 전환 효과
  - 버튼 호버 효과
  - 섹션 스크롤 애니메이션
  - 부드러운 트랜지션

- [ ] 타이포그래피 최적화
  - 제목 계층 구조 (h1, h2, h3)
  - 일관된 줄 높이 (line-height)
  - 적절한 문자 간격

### 산출물
- 모든 디바이스에서 작동하는 반응형 디자인
- 향상된 UI/UX
- 접근성 표준 준수

---

## Phase 6: 성능 최적화 및 SEO (1-2일)

### 목표
검색 엔진 최적화 및 성능 개선

### 작업 항목
- [ ] SEO 최적화
  - Meta 태그 작성 (description, keywords, og:image 등)
  - Sitemap 작성 (선택사항)
  - robots.txt 작성 (선택사항)
  - Open Graph 메타 태그 추가

- [ ] 성능 최적화
  - 이미지 포맷 최적화 (WebP 사용)
  - 이미지 압축 (TinyPNG, ImageOptim)
  - CSS/JavaScript 최소화 (선택사항)
  - Lighthouse 점수 개선 (90 이상 목표)

- [ ] 브라우저 호환성 테스트
  - Chrome 테스트
  - Firefox 테스트
  - Safari 테스트
  - Edge 테스트 (선택사항)

- [ ] 크로스 브라우저 CSS 최적화
  - 벤더 프리픽스 확인
  - CSS Grid/Flexbox 호환성 확인

- [ ] 분석 도구 설정 (선택사항)
  - Google Analytics 추가
  - 방문자 추적 설정

### 산출물
- 최적화된 메타 태그
- 개선된 성능 메트릭
- 모든 주요 브라우저에서 검증

---

## Phase 7: 배포 및 유지보수 (1일)

### 목표
웹사이트 배포 및 라이브 환경 구성

### 작업 항목
- [ ] GitHub 저장소 설정
  - Git 초기화 (필요시)
  - `.gitignore` 파일 생성
  - 모든 파일 커밋 및 푸시

- [ ] 배포 플랫폼 선택 및 배포
  - **GitHub Pages**: GitHub 저장소 설정에서 Pages 활성화
  - **또는 Vercel**: GitHub 연동 후 자동 배포
  - **또는 Netlify**: GitHub 연동 후 자동 배포

- [ ] 커스텀 도메인 설정 (선택사항)
  - 도메인 레지스트라 구매
  - DNS 설정 (CNAME, A 레코드)
  - HTTPS 활성화

- [ ] 배포 후 최종 검증
  - 라이브 사이트 기능 테스트
  - 모든 링크 확인
  - 모바일 반응형 재검증
  - 성능 메트릭 확인

- [ ] 모니터링 및 유지보수 설정
  - Google Analytics 데이터 검토 방법
  - 콘텐츠 업데이트 주기 결정
  - 정기 점검 계획

- [ ] 배포 자동화 설정 (선택사항)
  - GitHub Actions 설정
  - 자동 배포 파이프라인 구성

### 산출물
- 라이브 배포된 웹사이트
- 커스텀 도메인 (선택사항)
- 배포 문서

## 기술 스택 및 도구

- **HTML5**: 시맨틱 마크업
- **CSS3 + Tailwind CSS**: 스타일링 및 반응형 디자인
- **Vanilla JavaScript**: 상호작용 및 동적 기능
- **Git/GitHub**: 버전 관리
- **배포 플랫폼**: GitHub Pages / Vercel / Netlify
- **개발 도구**: 
  - 브라우저 DevTools
  - VS Code 또는 선호하는 에디터
  - 로컬 HTTP 서버 (Python 또는 Node.js)
