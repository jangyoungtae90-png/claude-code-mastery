# 개발자 웹 이력서 포트폴리오

> 개발자 경력, 기술, 프로젝트를 소개하는 **반응형 웹 포트폴리오** 사이트

## 🎯 프로젝트 개요

이 프로젝트는 개발자의 웹 이력서/포트폴리오 사이트입니다. 순수 HTML, CSS(Tailwind), JavaScript로 구축된 정적 사이트로, 빠른 로딩 속도와 뛰어난 SEO 성능을 제공합니다.

### 주요 특징
- ✨ **현대적 디자인**: Tailwind CSS 기반의 깔끔하고 전문적인 UI
- 📱 **반응형 디자인**: 모바일, 태블릿, 데스크톱 모든 기기에 최적화
- ⚡ **빠른 성능**: 빌드 단계 없이 CDN 기반 리소스로 즉시 로딩
- ♿ **접근성**: 시맨틱 HTML과 WCAG 표준 준수
- 🌙 **다크 모드**: 사용자 선호도에 따른 테마 지원 (Phase 5)

## 📁 프로젝트 구조

```
.
├── index.html              # 메인 HTML 파일

│   ├── images/             # 프로필 사진, 프로젝트 스크린샷

├── CLAUDE.md               # Claude Code 개발 가이드

```

## 🚀 빠른 시작

### 로컬 개발 서버 실행

빌드 단계가 필요 없습니다. 다음 중 하나를 선택해 로컬 서버를 실행하세요:

**Python 3**
```bash
python3 -m http.server 8000
```

**Python 2**
```bash
python -m SimpleHTTPServer 8000
```

**Node.js** (http-server 설치됨)
```bash
npx http-server
```

브라우저에서 `http://localhost:8000` 접속

### 브라우저 테스트
- Chrome, Firefox, Safari, Edge에서 테스트
- DevTools (F12)의 Device Toolbar로 모바일 반응형 테스트
- 실제 기기에서 최종 검증

## 📚 개발 가이드

### 주요 섹션

| 섹션 | 설명 |
|------|------|
| **Navigation** | 고정/스티킹 헤더 + 부드러운 스크롤 네비게이션 |
| **Hero** | 프로필 이미지, 인사말, CTA 버튼 |
| **About** | 전문가 요약 및 소개 |
| **Experience** | 직무 경력 타임라인 |
| **Skills** | 기술 스택 및 숙련도 |
| **Projects** | 포트폴리오 프로젝트 |
| **Contact** | 연락처 정보 및 소셜 링크 |

### 스타일링

- **Tailwind CSS**: 유틸리티-퍼스트 스타일링 (CDN 사용)
- **반응형 설계**: Mobile-first 접근
  - 기본: 모바일 (320px+)
  - `md:` 접두사: 태블릿/노트북 (768px+)
  - `lg:` 접두사: 큰 화면 (1024px+)

### JavaScript 기능

- 부드러운 섹션 스크롤
- 활성 네비게이션 링크 하이라이트
- 모바일 햄버거 메뉴 토글
- 스크롤 애니메이션 (AOS 라이브러리)

## 🎨 커스터마이징

### 콘텐츠 수정
`index.html`에서 직접 편집:
- 이름, 직급, 소개말
- 경력 정보 (회사, 직급, 기간, 성과)
- 기술 스택
- 프로젝트 정보
- 연락처

### 이미지 추가
1. 이미지를 `assets/images/` 폴더에 저장
2. `index.html`에서 경로 지정: `<img src="assets/images/photo.jpg" alt="설명">`
3. 반드시 alt 텍스트 추가 (접근성)

### 색상 변경
`index.html`의 Tailwind 설정에서:
```javascript
colors: {
    primary: '#4F46E5',    // 기본 색상
    secondary: '#0EA5E9',  // 보조 색상
}
```

## 🛠️ 개발 워크플로우

### 새로운 섹션 추가
1. `index.html`에 `<section>` 요소 추가 (고유한 `id` 필요)
2. 시맨틱 마크업 작성 (h2, h3, ul, li 등)
3. Tailwind 클래스로 스타일링 (반응형 접두사 활용)
4. 네비게이션 링크 추가
5. 필요하면 JavaScript 상호작용 추가

## 📦 배포

### GitHub Pages 배포 (무료)
1. GitHub 저장소 생성
2. 모든 파일 커밋 및 푸시
3. 저장소 Settings → Pages → GitHub Pages 활성화
4. `main` 브랜치 선택 및 저장

### Vercel 배포 (무료)
1. GitHub 저장소 연동
2. vercel.com 로그인
3. 새 프로젝트 추가
4. 자동 배포 (푸시할 때마다)

### Netlify 배포 (무료)
1. GitHub 저장소 연동
2. netlify.com 로그인
3. 새 사이트 생성
4. 자동 배포 설정
